PolicyGovernor — How Every HR Decision Is Audited

Unlike typical HR SaaS, the HR-LLM here is sealed and has no direct commit authority. An independent PolicyGovernor gates all operations with role-based access control, purpose limits, fairness checks, and minimal disclosure rules. Hard violations cannot be overridden—even by humans. The table below shows actual results from running the OperationActor four times: HR-LLM proposes → PolicyGovernor reviews → phase gate → human approval (if needed) → commit or hold.

What This Guarantees

  • Protected attributes (age, gender, nationality, health, marital, pregnancy) never ground an evaluation or appear in a report
  • Reports disclose only the columns your stated purpose permits
  • Role-based access rules cannot be bypassed by the LLM
  • High-stakes changes always require human approval
  • Every commit, hold, and approval is logged immutably
